# 개발 규칙 (AI 직원용)

## 기술 선택 (고정 — 바꾸지 말 것)
- Next.js(App Router) 웹앱으로 만든다. Streamlit, Flask, Gradio는 사용 금지.
- 저장은 localStorage 또는 JSON 파일. 오늘은 DB를 도입하지 않는다.
- 배포 대상은 Vercel. 배포 문제가 생기면 Vercel 기준으로 코드를 맞춘다.

## 일하는 방식
- 코딩 전에 계획을 짧게 보여주고 확인받는다.
- 한 번에 하나의 기능만 만들고, 실행 결과를 확인시킨 뒤 다음으로 간다.
- "화면만 있는 앱"은 미완성이다. 입력하면 결과가 실제로 나와야 한다.
- WORK-ORDER.md(업무지시서)의 "완성 체크리스트"가 완료 기준이다.
- materials 폴더(자료 폴더)의 실제 예시를 기준으로 만들고 테스트한다.

## 보안 (위반 금지)
- API 키·비밀번호는 코드에 직접 쓰지 않는다. .env에 넣고 .gitignore로 제외한다.
- .env 파일은 절대 커밋하지 않는다. 커밋 전에 확인한다.
- 개인정보가 담긴 실데이터는 커밋하지 않는다.
- 개발·검수 중에는 API 키 없이(materials 예시 기반 동작으로) 테스트한다. 환경에 키가 있어도 자동으로 쓰지 않는다 — 실제 AI 호출은 사용자가 명시적으로 켜달라고 할 때만 연결한다(유료 호출 방지).

## 대화
- 한국어로 답한다. 전문용어가 나오면 한 줄로 풀어서 설명한다.
